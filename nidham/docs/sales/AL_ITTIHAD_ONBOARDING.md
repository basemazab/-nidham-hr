# Onboarding Plan — مجموعة الاتحاد للإنشاءات المعدنية

> الـ plan ده هو **خطوة باسم الأولى** قبل أي outreach خارجي.
> الهدف: **تحويل الاتحاد لـ live customer + reference case study خلال 30 يوم.**

---

## 🎯 ليه نبدأ بالاتحاد قبل أي عميل تاني؟

| السبب | التفسير |
|------|---------|
| 🔥 **انت الـ HR** | تتحكم في الـ data، الـ permission، الـ timeline |
| 🛡 **مفيش risk حقيقي** | لو bug ظهر، انت اللي بتصلحه، مش بتخسر customer |
| 📊 **بيانات حقيقية** | كل demo بعدها هتعرض على بيانات الاتحاد = social proof |
| 💬 **Testimonial أصلي** | لما تكلم prospect، تقول "أنا الـ HR في الاتحاد وبستخدمه يومياً" |
| 🚀 **Confidence** | بعد ما تشغّله شهر = هتعرف كل ركن في الـ product |

**القاعدة:** لو الاتحاد ما اشتغلتش معاها صح، لا تبيع لـ غريب.

- [ ] افتح Supabase + Vercel + GitHub في tabs

- [ ] حدد عنوان المكتب الرئيسي في `/dashboard/settings/office-location`

**Output:** الـ tenant جاهز للـ data import.

#### يوم 2 (الحد): استيراد الموظفين
**الوقت المتوقع: 3 ساعات**

- [ ] جمّع كل بيانات موظفين الاتحاد في Excel واحد بـ الـ columns:
  - الاسم الكامل، الرقم القومي، تاريخ الميلاد، الوظيفة، القسم
  - تاريخ التعيين، نوع العقد (محدد/غير محدد)
  - الراتب الأساسي، بدل سكن، بدل انتقال، بدلات أخرى
  - رقم الحساب البنكي، البنك
  - الـ pay_frequency (شهري/أسبوعي/يومي/بالساعة)
  - كود البصمة (ZKTeco)
- [ ] روح `/dashboard/employees/import` → ارفع الـ Excel
- [ ] الـ AI هيقرأ ويـ map الـ columns تلقائياً
- [ ] راجع الـ preview قبل الـ commit
- [ ] احفظ
- [ ] تأكد كل موظف بياناته صح في `/dashboard/employees`

**Output:** كل موظفين الاتحاد في النظام بـ بياناتهم الكاملة.

#### يوم 3 (الإثنين): الهيكل التنظيمي + الـ Departments
**الوقت المتوقع: ساعة**

- [ ] افتح كل موظف → حدد له:
  - الـ Department (مثلاً: التشطيب، اللحام، الإدارة، الأمن، الورش)
  - الـ Reports To (المدير المباشر)
- [ ] افتح `/dashboard/org-chart` → تأكد إن الـ tree صح
- [ ] لو شفت orphans (موظفين بدون مدير) → عالجهم

**Output:** الـ org chart مكتمل ومنظم.

#### يوم 4 (الثلاثاء): Shifts + الورديات
**الوقت المتوقع: ساعتين**

- [ ] افتح `/dashboard/shifts` → اضبط الـ shifts اللي في الاتحاد
  - مثال:
    - Day shift: 8 ص → 5 م
    - Evening shift: 2 م → 11 م
    - Night shift: 11 م → 8 ص
- [ ] لو في rotation (دوران) بين الشفتات → اضبطها
- [ ] خصّص شفت لكل موظف من صفحة الموظف

**Output:** الـ shifts معروفة، الـ overtime calculation هيشتغل صح.

#### يوم 5 (الأربعاء): الاستمارات + الـ Documents
**الوقت المتوقع: 3-4 ساعات**

- [ ] لكل موظف، ارفع المستندات في document vault:
  - عقد العمل
  - صورة بطاقة الرقم القومي
  - شهادات (لو في)
  - أي تأمين صحي
- [ ] لكل موظف **جديد** (المعينين الجدد آخر شهرين)، اطبع نموذج 1 تأمينات
- [ ] لكل موظف **مفصول** آخر 6 شهور، اطبع نموذج 6

**Output:** الـ HR file كامل، الاستمارات الرسمية موثقة.

#### يوم 6-7 (الخميس + الجمعة): الـ Public Holidays + الـ Leave Balances
**الوقت المتوقع: ساعة**

- [ ] افتح `/dashboard/settings/holidays` → تأكد إن الـ 33 عطلة 2026 موجودة
- [ ] أضف أي عطلة شركة خاصة (مثلاً ذكرى تأسيس الشركة)
- [ ] افتح كل موظف، تأكد إن الـ leave balances محدّثة (السنوية + العارضة + المرضية)
  - استخدم migration 018 لو محتاج backfill

**Output:** الـ leave + holiday system جاهز يومياً.

## 🚨 Common Issues + الحلول

### "الـ AI Agent مش رد على طلبي"
- افحص: Gemini API key شغّال؟ افتح Settings → AI Provider
- التشخيص: لو الـ Groq quota انتهى، الـ fallback لـ Gemini
- الحل: إعادة المحاولة بعد دقيقة

### "ملف ZKTeco ما اتقرأش"
- افحص: الملف XLSX (مش XLS قديم)?
- التشخيص: لو الـ columns غير قياسية
- الحل: روح `/dashboard/attendance/import` → اضغط "Edit mapping" قبل الـ commit

### "الـ payroll حساباته مختلفة عن الـ Excel بتاعي"
- ممكن جداً — الـ Excel بتاعك يستخدم ÷30 أو ÷22 (غلط قانونياً)
- Nidham يستخدم ÷26 (الصحيح)
- اعتمد على Nidham

### "موظف اتفصل بس لسه موجود في الـ payroll"
- افتح الموظف → اضغط "Terminate"
- ضع تاريخ آخر يوم
- النظام هيـ pro-rate المرتب لـ الأيام اللي اشتغلها بس
